# Contributing

We welcome contributions to the Shopping List App!

## Getting Started

1.  Fork the repository.
2.  Follow the [Setup Guide](SETUP.md) to get your local environment running.

## Development Workflow

1.  Create a new branch for your feature or bugfix:
    ```bash
    git checkout -b feature/my-new-feature
    ```
2.  Make your changes.
3.  Ensure code style consistency (the project uses ESLint).
    ```bash
    npm run lint
    ```
4.  Commit your changes with clear messages.

## Pull Request Process

1.  Push your branch to your fork.
2.  Open a Pull Request against the `main` branch.
3.  Describe your changes and link any related issues.
4.  Wait for code review and address any feedback.

## Code of Conduct

Please be respectful and considerate in all interactions.
